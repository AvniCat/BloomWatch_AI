# Data

Per repo policy, **large raw datasets are not committed here**. Only the small
modelling-ready tables are included so the notebooks/scripts run out of the
box:

- **`revised_master_dataset.csv`** (~70 KB) — 552 rows × 23 columns. **Training-ready.**
  Extends the wide table with two documented-HAB-event columns
  (`hab_event_documented`, `hab_events_last_24mo`) and a pre-computed combined
  label (`bloom_or_documented`). Built by `code/pipelines/12_build_revised_master.py`.
  **This is the file the training scripts use by default.**
- `dataset_merged_india_monthly_wide.csv` (~65 KB) — legacy wide table.
  552 rows × 19 columns. Retained for backward compatibility.
- `hab_events_india.csv` — 9 documented HAB events (Kerala + Karnataka),
  extracted from CMFRI Annual Reports and published Indian HAB literature.
  Built by `code/pipelines/11_extract_hab_events.py`.

Everything else is regenerated by running the scripts in `code/pipelines/`.

## Column reference — `dataset_merged_india_monthly_wide.csv`

Identifier columns: `year, month, region`.

MODIS SST columns (from L3m monthly, degC): `sst_mean, sst_min, sst_max, sst_std, sst_n_pixels`.

MODIS Chlorophyll-a (from L3m monthly, mg/m³): `chlor_a_mean, chlor_a_min, chlor_a_max, chlor_a_std, chlor_a_n_pixels`.

IMD rainfall: `rainfall_mm_total_mean, rainfall_mm_total_min, rainfall_mm_total_max, rainfall_mm_max_daily, rainy_days_mean, rainfall_n_cells`.

Bloom label used in training scripts: derived at load time as `bloom = 1 if chlor_a_mean > 2 mg/m³ else 0`.
